Access Control System

A comprehensive multi-floor access control system that implements secure card management, floor access policies, and detailed audit logging. This system is designed to provide controlled access to different security levels within a facility.

Core Features

  • Multi-level Security: Different policies for low, medium, and high security floors
  • Time-based Access Control: Enforced time restrictions for different security levels
  • Audit Trail: Comprehensive logging of all access attempts and card management operations
  • Card Management: Creation, modification, and revocation of access cards
  • Encrypted IDs: Secure card identifiers with time-based encryption

Design Patterns

This system implements several design patterns to achieve flexibility, maintainability, and extensibility:

  • Factory Pattern: Used for creating different types of access cards
  • Singleton Pattern: Used for the audit logging system to ensure a single logging instance
  • Decorator Pattern: Used to enhance the audit logging functionality

Package Organization

  • com.camt.dii.secure - Core application components
  • com.camt.dii.secure.access - Floor access management and policies
  • com.camt.dii.secure.card - Card management and permissions
  • com.camt.dii.secure.card.factory - Card creation factories
  • com.camt.dii.secure.audit - Audit logging system
  • com.camt.dii.secure.audit.decorator - Enhanced audit logging decorators
  • com.camt.dii.secure.service - Access control services

OOP Principles Demonstrated

  • Encapsulation: Access cards encapsulate access rights and identifiers
  • Inheritance: Service hierarchies and permission implementations
  • Polymorphism: Floor access policies with different implementations
  • Abstraction: Interfaces for permissions and access control
